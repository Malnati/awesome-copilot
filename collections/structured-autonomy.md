## How It Works

Structured Autonomy e um workflow de tres fases projetado para maximizar o valor que voce obtem do desenvolvimento assistido por AI enquanto mantem requests premium baixos. O sistema segue um principio simples: **use modelos premium com parcimonia para pensar, use modelos baratos abundantemente para fazer**.

### The Workflow

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│    sa-plan      │ ──▶ │   sa-generate   │ ──▶ │   sa-implement  │
│  (1 request)    │     │   (1 request)   │     │  (many requests)│
│  Premium Model  │     │  Premium Model  │     │   Cheap Model   │
└─────────────────┘     └─────────────────┘     └─────────────────┘
```

**Phase 1: Plan** — Voce descreve o que deseja construir. O planning agent pesquisa seu codebase, le documentacao e decompõe sua feature em commits logicos. Output: `plans/{feature-name}/plan.md`


```
/plan Add a user profile page to this application that allows the user to view and edit their profile information. 
```

**Phase 2: Generate** — O generator pega seu plano e produz instrucoes completas de implementacao prontas para copiar e colar com blocos de codigo completos. Sem placeholders, sem TODOs — apenas codigo funcional baseado nos padroes reais do seu codebase. Output: `plans/{feature-name}/implementation.md`

```
/generate #plan.md
```

**Phase 3: Implement** — O implementation agent segue as instrucoes geradas passo a passo, marcando itens conforme avanca e parando em limites de commit definidos para voce verificar e commitar.

```
/implement #implementation.md
```

A implementacao para apos concluir um commit, permitindo que voce verifique as mudancas e depois prossiga. Ela retoma automaticamente do proximo passo.

### One Branch, One PR

Cada workflow opera em uma unica feature branch direcionada a um unico pull request. Etapas de implementacao mapeiam diretamente para commits, oferecendo:

- **Historico git limpo** com commits logicos e revisaveis
- **Checkpoints naturais** onde voce verifica se o codigo funciona antes de seguir
- **Rollback facil** se algo der errado — basta resetar para o ultimo commit valido

### Why This Approach Works

**Eficiencia de custo** — Modelos premium (Claude Opus, GPT-5.1-Codex) sao caros, mas excelentes em raciocinio e planejamento. Voce os usa exatamente duas vezes: uma para entender sua solicitacao e outra para gerar a implementacao. O trabalho de codificacao real usa modelos gratuitos que podem iterar quantas vezes forem necessarias.

**Melhor qualidade de codigo** — Como a fase de planejamento pesquisa seu codebase primeiro, o codigo gerado segue seus padroes existentes, convencoes de nomenclatura e arquitetura. Voce nao precisa lutar contra sugestoes de AI que nao se encaixam no seu projeto.

**Engajamento do desenvolvedor** — A implementacao passo a passo com pontos manuais de commit mantem voce no loop. Voce revisa e testa mudancas reais em cada etapa, nao fica olhando para um diff enorme no final se perguntando o que aconteceu.

**Workflow familiar** — Isto nao e uma nova forma de trabalhar — e seu workflow git existente com assistencia de AI. Feature branches, commits incrementais, PR reviews. A AI cuida das partes tediosas enquanto voce mantem controle do processo.
