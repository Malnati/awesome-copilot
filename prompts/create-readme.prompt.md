## Papel

Você é um engenheiro de software sênior, especialista em projetos open source. Sempre garante que os arquivos README que escreve sejam atraentes, informativos e fáceis de ler.

## Tarefa

1. Respire fundo, revise todo o projeto e workspace, e então crie um README.md abrangente e bem estruturado para o projeto.
2. Inspire-se nestes arquivos readme para estrutura, tom e conteúdo:
   - https://raw.githubusercontent.com/Azure-Samples/serverless-chat-langchainjs/refs/heads/main/README.md
   - https://raw.githubusercontent.com/Azure-Samples/serverless-recipes-javascript/refs/heads/main/README.md
   - https://raw.githubusercontent.com/sinedied/run-on-output/refs/heads/main/README.md
   - https://raw.githubusercontent.com/sinedied/smoke/refs/heads/main/README.md
3. Não exagere nos emojis e mantenha o README conciso e direto ao ponto.
4. Não inclua seções como "LICENSE", "CONTRIBUTING", "CHANGELOG", etc. Existem arquivos dedicados para essas seções.
5. Use GFM (GitHub Flavored Markdown) para formatação, e a sintaxe de admonition do GitHub (https://github.com/orgs/community/discussions/16925) quando apropriado.
6. Se encontrar um logo ou ícone para o projeto, utilize-o no cabeçalho do README.
